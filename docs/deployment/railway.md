# Деплой на Railway

Railway — самый быстрый путь от кода до публичного HTTPS-эндпоинта, который нужен МАКС для вебхуков. На бесплатном тарифе можно держать небольшие проекты, а для продакшена удобен платный план с автоматическими деплоями из GitHub.

## Чек-лист подготовки

- Аккаунт на [railway.app](https://railway.app)
- Репозиторий на GitHub с этим проектом
- Действующий токен МАКС в переменной `MAX_TOKEN`

## Шаги

### 1. Создать проект

В консоли Railway: **New Project → Deploy from GitHub repo**. Выбрать репозиторий с ботом. Railway сам определит, что это Python, и запустит установку зависимостей из `requirements.txt`.

### 2. Указать команду запуска

В настройках сервиса найдите раздел **Settings → Deploy → Start Command** и впишите:

```bash
uvicorn main:app --host 0.0.0.0 --port $PORT
```

Переменная `$PORT` подставляется Railway автоматически.

### 3. Прокинуть переменные окружения

В разделе **Variables** добавьте:

| Переменная   | Значение                          |
|--------------|-----------------------------------|
| `MAX_TOKEN`  | Токен из личного кабинета МАКС    |

После сохранения Railway автоматически перезапустит сервис.

### 4. Получить публичный домен

В разделе **Settings → Networking** нажмите **Generate Domain**. Railway выдаст URL вида `https://max-bot-gemini.up.railway.app`.

Проверьте, что бот живой:

```bash
curl https://max-bot-gemini.up.railway.app/health
# {"ok":true}
```

### 5. Зарегистрировать вебхук в МАКС

В личном кабинете разработчика МАКС укажите URL вебхука:

```
https://max-bot-gemini.up.railway.app/webhook
```

После сохранения отправьте боту `/start` в МАКС — должно прийти приветствие с кнопками.

## Журналы

Логи доступны в разделе **Deployments → View Logs**. Бот пишет туда:

- `INCOMING_UPDATE` — полный JSON каждого входящего обновления
- `MAX <url> -> <status> <body>` — результат каждого исходящего запроса

Если бот не отвечает, первое, что стоит проверить — приходят ли в лог строки `INCOMING_UPDATE`. Если их нет, проблема на стороне регистрации вебхука. Если есть, но в ответ от МАКС летит `4xx` или `5xx`, смотрим тело в логе.

## Типичные проблемы

**Бот молчит на `/start`.** Чаще всего — не задан `MAX_TOKEN`. Бот в этом случае возвращает `500` на `/webhook`, и в логах будет `MAX_TOKEN is missing`. Решение: добавить переменную и подождать перезапуска.

**Постоянные `500` от платформы.** В логе видны строки `MAX ... -> 401 ...`. Это значит, что токен задан, но недействителен или отозван. Сгенерируйте новый в личном кабинете и обновите переменную.

**Деплой собирается, но контейнер падает.** Откройте лог сборки и проверьте, что в `requirements.txt` нет конфликтующих версий. Railway использует pip и кеширует зависимости — изредка помогает форс-передеплой через **Redeploy**.

## Стоимость

Сам бот ест очень мало: один-два мегабайта памяти на стартe и десятки миллисекунд CPU на обновление. На стартовом платном тарифе Railway ($5/мес) такой бот занимает ничтожную долю квоты — остаётся место под несколько подобных проектов.
