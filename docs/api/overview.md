# Обзор API

Бот взаимодействует с MAX Platform API в обе стороны:

- **Входящий канал** — платформа МАКС отправляет на наш `/webhook` обновления о новых сообщениях и нажатиях кнопок.
- **Исходящий канал** — бот вызывает методы платформы, чтобы отправить сообщение пользователю или ответить на нажатие кнопки.

!!! note "Источник сведений"
    Описание формата ниже реконструировано по фактическому поведению API и коду бота. При официальных обновлениях документации МАКС эти страницы стоит сверять с источником.

## Базовые параметры

| Параметр | Значение                       |
|----------|--------------------------------|
| Базовый URL | `https://platform-api.max.ru` |
| Формат   | JSON                            |
| Кодировка | UTF-8                          |
| Авторизация | Заголовок `Authorization: <MAX_TOKEN>` |

Токен выдаётся в личном кабинете разработчика МАКС и хранится в переменной окружения `MAX_TOKEN`. В каждом исходящем запросе он передаётся как есть, без префикса `Bearer`.

## Эндпоинты, которые использует бот

| Направление | Метод | URL                                       | Назначение                          |
|-------------|-------|-------------------------------------------|--------------------------------------|
| Входящий    | `POST` | `<ваш-домен>/webhook`                     | Получение обновлений от МАКС         |
| Исходящий   | `POST` | `https://platform-api.max.ru/messages?chat_id={chat_id}` | Отправка сообщения в чат |
| Исходящий   | `POST` | `https://platform-api.max.ru/answers?callback_id={callback_id}` | Ответ на нажатие inline-кнопки |

## Типы обновлений

Платформа МАКС в текущей версии бота присылает обновления двух типов:

- `message_created` — пользователь отправил сообщение в чат.
- `message_callback` — пользователь нажал инлайн-кнопку.

Подробный формат полей разнесён по двум следующим разделам:

- [Входящие обновления](webhook.md)
- [Исходящие запросы](outgoing.md)

## Логирование

Каждое входящее обновление пишется в лог целиком:

```python
logging.info("INCOMING_UPDATE: %s", json.dumps(body, ensure_ascii=False))
```

Каждый исходящий запрос логируется вместе с HTTP-кодом и телом ответа:

```python
logging.info("MAX %s -> %s %s", url, resp.status_code, resp.text)
```

Этого достаточно, чтобы при инциденте восстановить картину по логам без подключения отладчика.
