# Входящие обновления

Платформа МАКС отправляет обновления методом `POST` на эндпоинт `/webhook` бота. Тело запроса — JSON. Бот всегда отвечает кодом `200 OK` и телом `{"ok": true}`, даже если обновление не распознано: это снижает риск повторных доставок.

## Общая структура

```json
{
  "update_type": "message_created",
  "message": { },
  "callback": { }
}
```

Какие поля присутствуют, зависит от значения `update_type`. В коде бот также допускает поле `type` как синоним `update_type` — оставлено для совместимости с возможными расхождениями версий платформы.

## update_type: `message_created`

Приходит, когда пользователь отправил боту текстовое сообщение.

### Пример

```json
{
  "update_type": "message_created",
  "message": {
    "recipient": {
      "chat_id": 1234567890
    },
    "body": {
      "text": "/start"
    }
  }
}
```

### Поля

| Путь                              | Тип    | Описание                                    |
|-----------------------------------|--------|---------------------------------------------|
| `update_type`                     | string | Тип обновления, здесь `"message_created"`   |
| `message.recipient.chat_id`       | int    | Идентификатор чата, в который пришло сообщение |
| `message.body.text`               | string | Текст сообщения от пользователя             |

### Как бот это обрабатывает

```python
if update_type == "message_created":
    message = body.get("message") or {}
    recipient = message.get("recipient") or {}
    chat_id = recipient.get("chat_id")
    text = ((message.get("body") or {}).get("text") or "").strip().lower()

    if chat_id and text in ("/start", "/старт"):
        send_message(chat_id, START_TEXT, with_buttons=True)
```

Реакция предусмотрена только на `/start` и `/старт`. Любые другие тексты игнорируются.

## update_type: `message_callback`

Приходит, когда пользователь нажал инлайн-кнопку под сообщением бота.

### Пример

```json
{
  "update_type": "message_callback",
  "callback": {
    "callback_id": "cb-9f1c2e7a-7f3b",
    "payload": "about"
  }
}
```

### Поля

| Путь                | Тип    | Описание                                                       |
|---------------------|--------|----------------------------------------------------------------|
| `update_type`       | string | Тип обновления, здесь `"message_callback"`                     |
| `callback.callback_id` | string | Идентификатор нажатия, нужен для ответа через `/answers`    |
| `callback.payload`  | string | Произвольная строка, заданная при формировании клавиатуры      |

### Возможные значения `payload`

Бот распознаёт два значения:

- `about` — отправить текст про обучение
- `signup` — отправить текст про запись

Прочие значения игнорируются.

## Ошибки

| Код     | Когда                                 | Что делать                              |
|---------|---------------------------------------|------------------------------------------|
| `400`   | Тело запроса не парсится как JSON     | Проверить, что МАКС шлёт корректное тело |
| `500`   | Переменная окружения `MAX_TOKEN` не задана | Установить токен и перезапустить       |

## Что не используется в этой версии

Платформа МАКС умеет присылать и другие типы обновлений: события входа в чат, добавления участников, прикрепления медиа и так далее. В текущей сборке бота они не обрабатываются — приём расширяется добавлением новой ветки в обработчике `webhook` в `main.py`.
